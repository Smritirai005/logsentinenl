# Log Anomaly Detection — MLOps Project

Real-time log anomaly detection pipeline built on AWS using an LSTM Autoencoder.

## Architecture

```
HDFS Logs → Kinesis Stream → Lambda (log_processor)
                                     ↓
                          Lambda (anomaly_detector)
                                     ↓
                          SageMaker Endpoint (LSTM Autoencoder)
                                     ↓
                     CloudWatch Metrics + SNS Alerts + Grafana
```

## Quick Start

### 1. Clone and install dependencies

```bash
git clone <your-repo-url>
cd log-anomaly-detection
pip install -r requirements.txt
```

### 2. Configure environment

```bash
cp .env.example .env
# Edit .env with your AWS account ID, S3 bucket, SageMaker role, etc.
```

Update `config.yaml` with your AWS account ID and resource names.
conda activate log-anomaly

### 3. Preprocess data

Downloads `HDFS_2k.log` automatically, parses into sequences, saves to `data/processed/`.

```bash
python src/data_preprocessing.py
```

### 4. Train the model

```bash
python src/model_training.py
```

Saves model artifacts to `models/`: `lstm_autoencoder.h5`, `vocab.pkl`, `threshold.pkl`.

### 5. Explore in the notebook

```bash
jupyter notebook notebooks/01_EDA_and_Training.ipynb
```

### 6. Deploy AWS infrastructure

```bash
cd aws_infrastructure
# 3. Redeploy
.\deploy.ps1
```

Provisions: S3 bucket, Kinesis stream, Lambda functions, IAM roles, SNS topic, CloudWatch alarms.

### 7. Deploy model to SageMaker

```bash
python src/deploy_model.py
```

### 8. Monitor with Grafana

Import `monitoring/grafana_dashboard.json` into Grafana connected to your CloudWatch data source.

## CI/CD (GitHub Actions)

| Workflow | Trigger | What it does |
|----------|---------|--------------|
| `train.yml` | Weekly (Monday 00:00 UTC) | Re-downloads data, retrains model, saves artifacts to S3 |
| `deploy.yml` | Push to `main` | Deploys CloudFormation stack + Lambda code + SageMaker endpoint |

## Model

LSTM Autoencoder trained on HDFS log event sequences.

| Parameter | Value |
|-----------|-------|
| Sequence length | 10 |
| Embedding dim | 32 |
| LSTM units | 64 |
| Epochs | 50 (early stopping) |
| Anomaly threshold | 95th percentile of training reconstruction error |

Anomaly is flagged when the reconstruction error of a new sequence exceeds the threshold.

## Drift Detection

`src/drift_detection.py` computes KL divergence between the reference event distribution (training set) and the current window. When it exceeds `0.15` (configurable in `config.yaml`), a SNS alert is sent and a CloudWatch metric is emitted.

## Dataset

HDFS_2k.log — 2,000 lines from the Hadoop Distributed File System log dataset by [LogPAI / loghub](https://github.com/logpai/loghub).
